  <action>
Update `src/app/globals.css` to implement dark-first theme and hot pink brand accent.

**Changes to make:**

1. **Set :root variables to dark values directly** (no media query):
   ```
   --background: #0a0a0a;
   --foreground: #ededed;
   ```

2. **Remove the entire `@media (prefers-color-scheme: dark)` block.** The app is always dark.

3. **Add brand accent color variables in :root:**
   ```
   --brand: #F472B6;
   --brand-muted: #DB2777;
   --brand-foreground: #0a0a0a;
   ```
   - `--brand` is hot pink (#F472B6 / Tailwind pink-400) — the primary accent for buttons, links, active states
   - `--brand-muted` is a deeper pink (#DB2777 / Tailwind pink-600) — for hover states, borders, subtle backgrounds
   - `--brand-foreground` is near-black — text color ON pink backgrounds (for contrast)
   - Both pink values pass WCAG AA contrast against #0a0a0a background (5.4:1+)

4. **Add a plain `@theme` block** (separate from existing `@theme inline`) for brand color tokens:
   ```css
   @theme {
     --color-brand: var(--brand);
     --color-brand-muted: var(--brand-muted);
     --color-brand-foreground: var(--brand-foreground);
   }
   ```
   This generates utility classes: `bg-brand`, `text-brand`, `border-brand`, `bg-brand-muted`, `text-brand-foreground`, etc.
   
   CRITICAL: Do NOT add these inside `@theme inline` — the `inline` modifier breaks CSS variable runtime overrides (see research Pitfall 5).

5. **Keep the existing `@theme inline` block unchanged** — it handles font aliasing correctly.

6. **Keep the `body` rule** referencing `var(--background)` and `var(--foreground)`.

**Target final file:**
```css
@import "tailwindcss";

:root {
  --background: #0a0a0a;
  --foreground: #ededed;
  --brand: #F472B6;
  --brand-muted: #DB2777;
  --brand-foreground: #0a0a0a;
}

@theme {
  --color-brand: var(--brand);
  --color-brand-muted: var(--brand-muted);
  --color-brand-foreground: var(--brand-foreground);
}

@theme inline {
  --color-background: var(--background);
  --color-foreground: var(--foreground);
  --font-sans: var(--font-heebo), ui-sans-serif, system-ui, sans-serif, "Apple Color Emoji", "Segoe UI Emoji", "Segoe UI Symbol", "Noto Color Emoji";
  --font-mono: ui-monospace, "Cascadia Code", "Source Code Pro", Menlo, Consolas, "DejaVu Sans Mono", monospace;
}

body {
  background: var(--background);
  color: var(--foreground);
}
```
