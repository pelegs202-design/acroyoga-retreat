**2. Create src/components/admin/AdminPanel.tsx** ('use client'):

Tabbed admin panel with 4 tabs: Members, Challenge Signups, Workshop Bookings, Audit Log. Use simple state-driven tabs (no routing — all within Settings page).

Tab implementation: use useState for activeTab. Each tab renders its component lazily (only fetch data when tab is active). Use Framer Motion for tab content transitions (fadeUp pattern consistent with project).

On mount, fetch summary stats from multiple endpoints in parallel:
- GET /api/admin/members (for total count + new this week)
- GET /api/admin/challenge-signups (for active count)
- GET /api/admin/workshop-bookings (for pending count)

Pass stats to AdminSummaryStats. Pass members data to MemberTable.

Use useTranslations('admin') for all labels.

Styling: Dark theme consistent with existing Settings page. Admin section separated from NotificationPreferences by a thick border-t border-neutral-800 with mt-12 pt-8. Section title "Admin Panel" in text-2xl font-bold text-neutral-100.

Tab bar: flex gap-2, each tab is a button with text-neutral-400 hover:text-neutral-100, active tab gets text-brand border-b-2 border-brand.

**3. Create src/components/admin/AdminSummaryStats.tsx** ('use client'):

4 stat cards in a 2x2 grid (grid-cols-2 gap-4). Each card: bg-neutral-900 rounded-lg p-4 with label (text-sm text-neutral-400) and value (text-2xl font-bold text-neutral-100).

Stats:
- Total Members (count from members array)
- New This Week (filter members where createdAt > 7 days ago)
- Active Challenges (count from challenge signups where status='confirmed')
- Pending Workshops (count from workshop bookings where contactStatus='new' or null)

**4. Create src/components/admin/MemberTable.tsx** ('use client'):

Use @tanstack/react-table with:
- Columns: Name, Email, City, Role, Level, Status, Host (boolean badge), Joined (formatted date), Actions
- SortingState + getSortedRowModel for sortable columns
- ColumnFiltersState + getFilteredRowModel for column filters
- globalFilter + text input for text search across name+email

Above the table: text search input (w-full) + row of filter dropdowns for Status (active/suspended), City, Role (base/flyer/both), Level (beginner/intermediate/advanced). Use simple select elements styled with dark theme.

Add a "Hosts only" toggle button that filters to isJamHost=true (per user decision: dedicated hosts-only filtered view via toggle).

Table styling: Dark theme, bg-neutral-900 rounded-lg overflow-hidden. Header row bg-neutral-800 text-neutral-400 text-sm. Body rows border-b border-neutral-800 text-neutral-300. Hover: bg-neutral-800/50. Sortable headers show cursor-pointer and sort direction indicator (arrow up/down).

Status column: 'active' shows green dot + text, 'suspended' shows red dot + text.
Host column: boolean badge (pink "Host" badge if true, nothing if false).

Receives members as prop (fetched by AdminPanel parent). Also receives onMemberUpdate callback for refreshing after mutations.

**5. Create src/components/admin/MemberActionButtons.tsx** ('use client'):

Per-row action buttons. Receives member object + onUpdate callback.

Buttons:
- If status='suspended': "Approve" button (green-ish text) — calls PATCH with action='approve', instant (no confirm)
- If status='active': "Suspend" button (yellow text) — confirm dialog ("Are you sure you want to suspend {name}?"), then PATCH action='suspend'
- If isJamHost=false: "Grant Host" button — instant, PATCH action='grant-host'
- If isJamHost=true: "Revoke Host" button (red text) — confirm dialog ("Revoking host will cancel all their future jams. Continue?"), then PATCH action='revoke-host'
- "Delete" button (red, always present) — confirm dialog ("This will permanently delete {name}'s account. This cannot be undone."), then DELETE request
- "View Profile" link — opens /members/{userId} in new tab

Confirmation dialogs: Use window.confirm() for simplicity (no modal library needed for admin-only destructive actions). After successful mutation, call onUpdate() to refresh the member list.

Buttons use small text (text-xs), spaced in a flex gap-1 row. Destructive actions in red, constructive in green/blue, neutral in default.

Show loading state during API calls (disable buttons, show spinner or "..." text).

**4. Update src/app/[locale]/(app)/settings/page.tsx:**

CRITICAL: Admin check happens in the Server Component — not in a client component. This prevents any flash of admin content for non-admins.

Add imports:
```typescript
import { isAdminEmail } from '@/lib/admin-guard';
import { AdminPanel } from '@/components/admin/AdminPanel';
```

After the existing `const session = await getAuthSession()` and redirect check, add:
```typescript
const isAdmin = session ? isAdminEmail(session.user.email) : false;
```

In the return JSX, add the AdminPanel conditionally AFTER NotificationPreferences:
```tsx
<NotificationPreferences />
{isAdmin && (
  <>
    <div className="mt-12 border-t border-neutral-800 pt-8">
      <AdminPanel />
    </div>
  </>
)}
```

This ensures: (a) non-admins see zero HTML trace of admin panel, (b) no client-side flash, (c) admin check is server-authoritative.
