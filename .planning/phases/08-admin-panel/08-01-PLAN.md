**2. Create src/lib/admin-guard.ts:**

Shared admin identity check utility used by all admin API routes and the Settings page:
```typescript
import { auth } from '@/lib/auth';
import { headers } from 'next/headers';
import { NextResponse } from 'next/server';

export function getAdminEmails(): string[] {
  return (process.env.ADMIN_EMAILS ?? '').split(',').map(e => e.trim()).filter(Boolean);
}

export function isAdminEmail(email: string): boolean {
  return getAdminEmails().includes(email);
}

export async function getAdminSession() {
  const session = await auth.api.getSession({ headers: await headers() });
  if (!session) return { session: null, error: NextResponse.json({ error: 'Unauthorized' }, { status: 401 }) };
  if (!isAdminEmail(session.user.email)) {
    return { session: null, error: NextResponse.json({ error: 'Not found' }, { status: 404 }) };
  }
  return { session, error: null };
}
```

Use 404 (not 403) for non-admin requests to avoid leaking that the admin endpoint exists.

**2. PATCH + DELETE /api/admin/members/[id]/route.ts:**

PATCH accepts JSON body with `action` field. Validate with Zod:
```typescript
const schema = z.object({
  action: z.enum(['approve', 'suspend', 'grant-host', 'revoke-host']),
});
```

Action handling:
- `approve`: Set user.status = 'active'. Write audit log: action='approve_member', targetType='user'.
- `suspend`: Set user.status = 'suspended'. Write audit log: action='suspend_member'. Do NOT block login (per research recommendation — suspend = hidden from search + blocked from community APIs).
- `grant-host`: Set user.isJamHost = true. Write audit log: action='grant_host'.
- `revoke-host`: This is the complex cascade:
  1. Set user.isJamHost = false
  2. Find all future jams where hostId = this member (scheduledAt > now())
  3. For each future jam: find all confirmed attendees, bulk update jamAttendees.status to 'cancelled'
  4. For each confirmed attendee: call queuePushNotification(attendee.userId, 'jam_rsvp', 'Jam cancelled', `The jam at ${jam.location} was cancelled`, '/jams')
  5. Write audit log: action='revoke_host', metadata includes count of cancelled jams and notified attendees

DELETE: Deletes the user account. This cascades via ON DELETE CASCADE in schema. Validate the [id] param exists. Write audit log before deletion (action='delete_member', metadata includes user name+email for record).

All PATCH/DELETE routes call `getAdminSession()` first.
