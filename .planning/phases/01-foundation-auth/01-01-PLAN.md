  <action>
    Create `src/lib/db/index.ts`:
    - Import `neon` from `@neondatabase/serverless` and `drizzle` from `drizzle-orm/neon-http`
    - Create neon sql client with `process.env.DATABASE_URL!`
    - Export `db` instance from `drizzle(sql)`

    Create `src/lib/db/schema.ts`:
    - Import pg-core types from drizzle-orm
    - Define a `tosAcceptances` table for version-tracked TOS acceptance history: id (text PK), userId (text, not null), tosVersion (text, default "v1"), acceptedAt (timestamp, default now), ipAddress (text, nullable)
    - Add a comment noting Better Auth will generate user/session/account/verification tables with additionalFields

    Create `src/lib/auth.ts`:
    - Import betterAuth, drizzleAdapter, nextCookies, db, Resend
    - Configure Better Auth with:
      - database: drizzleAdapter(db, { provider: "pg" })
      - emailAndPassword: enabled, minPasswordLength: 8
      - sendResetPassword callback using Resend (void the await to prevent timing attacks). From address: "AcroYoga Academy <noreply@acro.academy>" — use env var RESEND_FROM_EMAIL if available, fallback to this default
      - session: expiresIn 30 days (60*60*24*30), updateAge 1 day, cookieCache enabled with 5-min maxAge
      - user.additionalFields: city (string, optional, input:true), role (string, optional, input:true — values: base/flyer/both validated in app layer), level (string, optional, input:true — values: beginner/intermediate/advanced), preferredLocale (string, optional, input:false), tosAcceptedAt (date, optional, input:false)
      - plugins: [nextCookies()] as last plugin
    - Export `auth`

    Create `src/lib/auth-client.ts`:
    - Import createAuthClient from "better-auth/react"
    - Create client with baseURL from NEXT_PUBLIC_APP_URL
    - Export authClient and destructured { signIn, signUp, signOut, useSession }

    Create `app/api/auth/[...all]/route.ts`:
    - Import auth from "@/lib/auth"
    - Export GET and POST as auth.handler

    Run `npx @better-auth/cli generate` to generate the auth schema SQL. Then run `npx @better-auth/cli migrate` (or use drizzle-kit push) to push the schema to the database. If DATABASE_URL is not set yet, document the command in a comment at the top of schema.ts for the user to run after setting up their database.

    IMPORTANT per research: additionalFields with enum-like values (role, level) should use string type and validate in application layer, NOT Postgres enums — Better Auth generate may not handle custom enums correctly.
  </action>
