**Create src/app/api/notifications/preferences/route.ts:**
- **GET handler:** Requires authenticated session. Looks up notificationPreferences by userId. If no row exists, return defaults: `{ pushEnabled: true, emailMarketing: true, whatsappEnabled: true, quietHoursStart: 22, quietHoursEnd: 8 }`.
- **PATCH handler:** Requires authenticated session. Accepts partial body with any of: pushEnabled, emailMarketing, whatsappEnabled, quietHoursStart, quietHoursEnd. Validate types (booleans for toggles, integers 0-23 for hours). Upsert into notificationPreferences (insert if not exists, update if exists — use Drizzle's `onConflictDoUpdate` on userId PK).
- **Side effects on PATCH:**
  - If whatsappEnabled changed to false: cancel all active WhatsApp drip enrollments for this user's leads. Query dripEnrollments where channel='whatsapp' and cancelledAt IS NULL, look up by userId or by matching the user's email/phone to lead records. Cancel with reason 'opted_out'.
  - If pushEnabled changed to false: optionally delete all pushSubscriptions for this user (or just let the preference flag block sending — preference check in cron is sufficient).
