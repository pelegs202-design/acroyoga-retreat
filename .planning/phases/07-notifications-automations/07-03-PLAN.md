**Create src/app/api/cron/whatsapp-drip/route.ts:**
- GET handler with CRON_SECRET Bearer token guard.
- `export const runtime = 'nodejs'`
- Query dripEnrollments: `WHERE channel = 'whatsapp' AND next_fire_at <= now() AND completed_at IS NULL AND cancelled_at IS NULL LIMIT 50`.
- For each due enrollment:
  1. Look up the sequence definition from DRIP_SEQUENCES[sequenceType].
  2. Get current step's template name from the sequence definition.
  3. Determine language code: enrollment.preferredLocale — 'he' for Hebrew, 'en_US' for English.
  4. Build bodyParams array: [recipientName] (first name personalization per locked decision).
  5. Parse metadata JSON if present for additional template params (e.g., cohort start date, challenge name).
  6. Call `sendWhatsAppTemplate({ to: recipientPhone, templateName, languageCode, bodyParams })`.
  7. On success: increment currentStep. If currentStep >= totalSteps, set completedAt=now(). Otherwise, calculate nextFireAt from sequence spacing (DRIP_SEQUENCES spacingDays array).
  8. On WhatsApp API error: log the error. Do NOT increment step. Set nextFireAt to 1 hour from now (retry). After 3 consecutive failures for the same enrollment (track via a simple retry counter in metadata), cancel the enrollment with reason 'error'.
- **Expiry check for wa_challenge_prepay:** Parse metadata.cohortStartDate. If the cohort start date has passed and enrollment is still active, send the "missed-it" template (last template in the sequence), set completedAt, and cancel with reason 'expired'.
- **STOP opt-out handling:** This is handled by Meta's side. When a user replies STOP, Meta marks the number as opted out and future sends will fail with a specific error. Catch this error and cancel the enrollment with reason 'opted_out'.
- Return JSON: `{ processed: N, sent: N, errors: N }`.
- Wrap everything in try/catch — return 200 on all outcomes to prevent Vercel retry storms.

**Create src/app/api/cron/jam-reminders/route.ts:**
- GET handler with CRON_SECRET Bearer token guard.
- `export const runtime = 'nodejs'`
- This cron runs twice daily (05:00 UTC = 08:00 Israel, 15:00 UTC = 18:00 Israel).
- Determine which reminder window this is based on current UTC hour:
  - If UTC hour is ~5: this is the "morning of" reminder (sessions happening today).
  - If UTC hour is ~15: this is the "day before" reminder (sessions happening tomorrow).
- Query jam_sessions with scheduledAt matching the target date. Join with jamAttendees where status='confirmed' to get all confirmed attendees.
- For each attendee:
  1. Look up their phone number from user table (or quizLeads if linked).
  2. Check notificationPreferences.whatsappEnabled — skip if false.
  3. Send WhatsApp template: 'session_reminder_he' (or _en) with params: [attendeeName, sessionDate, sessionTime, location].
  4. Also send email reminder (import Resend from auth.ts pattern — or use a shared resend instance). Use react-email SessionReminder template (created in Plan 04). If template not yet available, use a simple text email as fallback.
- Log results and return 200.

Note: Both email and WhatsApp reminders always send — per locked decision "Channel overlap: Both WhatsApp AND email always — redundancy intentional, no fallback logic."
