**api/quiz/events/route.ts** — POST endpoint for tracking quiz step events:
- Accept JSON body: `{ sessionId: string, quizType: string, questionId: string, answer?: string, eventType: string }`
- Validate with zod: sessionId required, quizType required, questionId required, eventType required (one of 'view' | 'answer' | 'abandon'), answer optional
- Insert into `quizEvents` table with `id` generated via `crypto.randomUUID()`
- Return `{ ok: true }` on success, `{ error: string }` on failure
- No auth required (quiz is open to anyone per CONTEXT.md)
- Import `db` from `@/lib/db` and `quizEvents` from `@/lib/db/schema`

**api/quiz/leads/route.ts** — POST endpoint for saving quiz completions:
- Accept JSON body: `{ sessionId: string, quizType: string, name: string, email: string, phone: string, answers: Record<string, string>, resultType?: string, city?: string }`
- Validate with zod: sessionId required, quizType required ('challenge' | 'workshop'), name min 2, email valid, phone matches `^\+[1-9]\d{7,14}$`, answers required (as JSON string), resultType optional, city optional
- Insert into `quizLeads` table with `id` generated via `crypto.randomUUID()`, stringify answers if not already string
- Return `{ ok: true, sessionId }` on success
- On duplicate sessionId (unique constraint violation, error code 23505), return `{ ok: true, sessionId }` (idempotent — the lead was already saved)
- No auth required
- Import `db` from `@/lib/db` and `quizLeads` from `@/lib/db/schema`
  </action>
