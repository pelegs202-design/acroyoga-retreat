<task type="auto">
  <name>Task 2: Create RSVP API with race-condition-safe transaction and waitlist promotion</name>
  <files>src/app/api/jams/[id]/rsvp/route.ts</files>
  <action>
Create `src/app/api/jams/[id]/rsvp/route.ts` with a POST handler.

**POST /api/jams/[id]/rsvp** — Join or cancel RSVP:
- Auth required (same pattern as other API routes)
- Extract `jamId` from route params (`params.id`)
- Parse body: `{ action: 'join' | 'cancel' }`

**Join flow (action === 'join'):**
1. Check user doesn't already have an active RSVP (status = 'confirmed' or 'waitlist') for this jam. If exists, return 409 `{ error: 'Already registered' }`
2. Use `db.transaction` with row-level lock (CRITICAL for race condition prevention):
   ```
   const result = await db.transaction(async (tx) => {
     // Lock the jam row to prevent concurrent RSVP race condition
     const [jam] = await tx.select().from(jamSessions).where(eq(jamSessions.id, jamId)).for('update');
     if (!jam) throw new Error('not_found');

     // Count confirmed attendees
     const [{ confirmedCount }] = await tx.select({ confirmedCount: count() }).from(jamAttendees)
       .where(and(eq(jamAttendees.jamId, jamId), eq(jamAttendees.status, 'confirmed')));

     const status = confirmedCount >= jam.capacity ? 'waitlist' : 'confirmed';

     await tx.insert(jamAttendees).values({
       id: randomUUID(),
       jamId,
       userId: session.user.id,
       status,
       joinedAt: new Date(),
     });

     return { status, confirmedCount: status === 'confirmed' ? confirmedCount + 1 : confirmedCount, waitlistPosition: status === 'waitlist' ? /* compute position */ undefined : undefined };
   });
   ```
3. If status is 'waitlist', compute waitlist position (count of waitlist entries with joinedAt before this one + 1)
4. Return `{ ok: true, status: result.status, waitlistPosition?: number }`

**Cancel flow (action === 'cancel'):**
1. Find user's active RSVP (status = 'confirmed' or 'waitlist')
2. If no active RSVP, return 404 `{ error: 'No active RSVP found' }`
3. **4-hour lock check:** Fetch the jam's scheduledAt. If `Date.now() > scheduledAt.getTime() - (4 * 60 * 60 * 1000)`, return 403 `{ error: 'Cannot cancel within 4 hours of jam start' }`. Server-side only — never trust client time.
4. Update the RSVP status to 'cancelled'
5. **Waitlist promotion (only if the cancelled RSVP was 'confirmed'):**
   - Find the first waitlisted attendee (ORDER BY joinedAt ASC LIMIT 1)
   - Update their status to 'confirmed'
   - Send promotion email via Resend:
     - Fetch promoted user's email from user table
     - Import Resend from existing pattern in `src/lib/auth.ts` (reuse RESEND_API_KEY env var and FROM_EMAIL pattern)
     - Subject: "A spot opened up for you!"
     - Body: "Great news — a spot opened in the jam on {date} at {location}. You're confirmed!"
     - AWAIT the send (unlike auth password reset which voids) — this is a business-critical notification. Log errors if send fails.
6. Return `{ ok: true, cancelled: true }`

Import `count`, `eq`, `and`, `asc` from 'drizzle-orm'. Import Resend and create a local instance (same pattern as auth.ts).
