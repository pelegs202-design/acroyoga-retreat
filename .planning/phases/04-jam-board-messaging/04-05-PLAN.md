**5. Create `src/components/messages/ChatThread.tsx`** — 'use client':
- **Full-screen iMessage-style layout** (per CONTEXT.md):
  - Back button at top (← or arrow) — navigates to `/messages`
  - Other user's name at top center
  - Message area fills remaining height (flex-grow, overflow-y-auto)
  - Input bar fixed at bottom (text input + send button)
- **Initial load:** Fetch messages from `GET /api/messages/{conversationId}` — loads last 50 messages
- **Scroll-to-bottom** on initial load and when new messages arrive
- **Polling for new messages (CRITICAL — no Ably):**
  ```typescript
  useEffect(() => {
    const interval = setInterval(async () => {
      const res = await fetch(`/api/messages/${conversationId}`);
      const data = await res.json();
      setMessages(data.messages);
    }, 3000); // Poll every 3 seconds
    
    return () => clearInterval(interval);
  }, [conversationId]);
  ```
  - Poll every 3 seconds when the chat is open
  - Compare incoming messages with current state — only update if there are new messages (to avoid unnecessary re-renders and scroll jumps)
  - Smart comparison: check if the latest message ID differs from the current latest
- **Load older messages:** On scroll to top, fetch older messages with `?before={oldestMessageSentAt}&limit=50`. Prepend to messages array. Maintain scroll position.
- **Send message:** POST to `/api/messages/{conversationId}` with `{ text }`.
  - Optimistic UI: append the message to the list immediately with a "sending" state
  - On success: update with server-confirmed data (id, sentAt)
  - On error: show error state on the message, allow retry
- **Input bar:** text input with send button (brand-colored arrow or "Send" text). Disabled when empty. Enter key sends (Shift+Enter for newline is optional — keep simple for V1, just Enter sends).

  const [unreadCount, setUnreadCount] = useState(0);
