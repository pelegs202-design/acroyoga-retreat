**1. Create `src/app/[locale]/(app)/jams/page.tsx`** — Server component:
- Auth guard via `getAuthSession()` from `@/lib/auth-guard`
- `setRequestLocale(locale)` at top (same pattern as all [locale] server components)
- Fetch initial jam data server-side (or pass to client component to fetch)
- Check if current user isJamHost (query user table for isJamHost flag)
- Render `<JamFeed>` passing isJamHost boolean and locale

**2. Create `src/components/jams/JamFeed.tsx`** — 'use client' component:
- Props: `isJamHost: boolean`, `locale: string`
- State: jams array (upcoming), past jams array, city filter, level filter, loading
- Fetch jams from GET /api/jams on mount and when filters change
- **Upcoming section:**
  - Vertical list of JamCard components, sorted by scheduledAt ASC (next jam first)
  - If isJamHost, show "Post a jam" button (Link to /jams/new) — styled with bg-brand text-brand-foreground
- **Filters:** City text input + Level select (same UX pattern as MemberFilters from Phase 3 — consistent UX per user decision)
  - City: free text input with onChange
  - Level: select with options: All, Beginner, Intermediate, Advanced
  - Filters update query params and refetch
- **Past section:** Below upcoming, in a `<details>` element (collapsed by default per discretion recommendation):
  - Summary text: "Past jams ({count})"
  - Expand reveals past jam cards (read-only, no RSVP buttons)
- Empty state: "No upcoming jams" message

**3. Create `src/components/jams/JamCard.tsx`** — 'use client' component:
- Props: jam object (id, scheduledAt, location, capacity, level, notes, host, confirmedCount, waitlistCount, userRsvp, attendees), onRsvpChange callback
- Dark card styling: `bg-neutral-900 border border-neutral-800 rounded-lg p-4`
- Layout:
  - **Top row:** Date/time (formatted for locale), level badge
  - **Middle:** Location text, host name + avatar (small)
  - **Attendees:** Stacked avatar pill — up to 4 overlapping circles (`-ml-2` pattern), then "+N more" text if more than 4
  - **Spots:** "{confirmedCount}/{capacity}" with visual indicator. Text changes color when full (red/amber)
  - **RSVP button:**
    - If userRsvp is null and spots available: "Join" button (bg-brand)
    - If userRsvp is null and full: "Join Waitlist" button (border-brand, outlined style)
    - If userRsvp.status === 'confirmed': "Cancel" button (border-red-500, outlined)
    - If userRsvp.status === 'waitlist': "Leave Waitlist" button (border-neutral-500)
    - **4-hour lock:** If within 4 hours of scheduledAt, hide/disable cancel button with tooltip "Cannot cancel within 4 hours"
    - Check: `Date.now() > new Date(jam.scheduledAt).getTime() - 4 * 60 * 60 * 1000`
  - RSVP action: POST to `/api/jams/${jam.id}/rsvp` with `{ action: 'join' }` or `{ action: 'cancel' }`
  - After RSVP action, call onRsvpChange to refetch the jam list
- Use CSS logical properties throughout (ms-, me-, ps-, pe-) for RTL support
  </action>
