<task type="auto">
  <name>Task 1: Extend Drizzle schema with Phase 4 tables</name>
  <files>src/lib/db/schema.ts</files>
  <action>
Add these to the existing schema.ts (after the reviews table, before relations):

1. **Add `isJamHost` column to user table:**
   - `isJamHost: boolean("is_jam_host").default(false).notNull()` — approved hosts only (Phase 8 admin panel manages this; for now set manually in DB)

2. **Add `jamSessions` table:**
   - `id: text("id").primaryKey()`
   - `hostId: text("host_id").notNull().references(() => user.id, { onDelete: "cascade" })`
   - `scheduledAt: timestamp("scheduled_at", { withTimezone: true }).notNull()` — MUST use `withTimezone: true` to prevent 4-hour cancellation lock timezone bugs
   - `location: text("location").notNull()` — free text
   - `capacity: integer("capacity").notNull()` — max spots
   - `level: text("level").notNull()` — 'beginner' | 'intermediate' | 'advanced' | 'all'
   - `notes: text("notes")` — optional description
   - `createdAt: timestamp("created_at").defaultNow().notNull()`
   - Indexes: `jam_sessions_scheduled_idx` on scheduledAt, `jam_sessions_host_idx` on hostId

3. **Add `jamAttendees` table:**
   - `id: text("id").primaryKey()`
   - `jamId: text("jam_id").notNull().references(() => jamSessions.id, { onDelete: "cascade" })`
   - `userId: text("user_id").notNull().references(() => user.id, { onDelete: "cascade" })`
   - `status: text("status").notNull()` — 'confirmed' | 'waitlist' | 'cancelled'
   - `joinedAt: timestamp("joined_at").defaultNow().notNull()` — FIFO ordering key for waitlist
   - Indexes: `jam_attendees_jam_idx` on jamId, `jam_attendees_user_idx` on userId

4. **Add `conversations` table:**
   - `id: text("id").primaryKey()`
   - `participantA: text("participant_a").notNull().references(() => user.id, { onDelete: "cascade" })`
   - `participantB: text("participant_b").notNull().references(() => user.id, { onDelete: "cascade" })`
   - participantA < participantB enforced at application layer (sorted IDs for deterministic lookup)
   - `createdAt: timestamp("created_at").defaultNow().notNull()`
   - `lastMessageAt: timestamp("last_message_at").defaultNow().notNull()`
   - Index: `conversations_participants_idx` on (participantA, participantB)

5. **Add `directMessages` table:**
   - `id: text("id").primaryKey()`
   - `conversationId: text("conversation_id").notNull().references(() => conversations.id, { onDelete: "cascade" })`
   - `senderId: text("sender_id").notNull().references(() => user.id, { onDelete: "cascade" })`
   - `text: text("text").notNull()`
   - `sentAt: timestamp("sent_at").defaultNow().notNull()`
   - Indexes: `direct_messages_conversation_idx` on conversationId, `direct_messages_sent_idx` on sentAt

6. **Add `conversationReads` table:**
   - `conversationId: text("conversation_id").notNull().references(() => conversations.id, { onDelete: "cascade" })`
   - `userId: text("user_id").notNull().references(() => user.id, { onDelete: "cascade" })`
   - `lastReadAt: timestamp("last_read_at").defaultNow().notNull()`
   - No explicit primary key — composite of (conversationId, userId). Use unique index or handle via application upsert.

7. **Add relations for all new tables:**
   - `jamSessionsRelations`: host (one user), attendees (many jamAttendees)
   - `jamAttendeesRelations`: jam (one jamSession), user (one user)
   - `conversationsRelations`: participantAUser (one user), participantBUser (one user), messages (many directMessages)
   - `directMessagesRelations`: conversation (one conversations), sender (one user)
   - Update `userRelations` to include: `jamSessionsHosted (many jamSessions)`, `jamAttendances (many jamAttendees)`, `conversationsA (many conversations via participantA)`, `conversationsB (many conversations via participantB)`

Import `integer` from `drizzle-orm/pg-core` (not yet imported in schema.ts).

NOTE: Define relations AFTER all table declarations to avoid forward-reference TypeScript errors (same pattern as Phase 3 decision 03-01).
