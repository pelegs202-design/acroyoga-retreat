<task type="auto">
  <name>Task 1: Extend schema.ts with Phase 4 tables and relations</name>
  <files>src/lib/db/schema.ts</files>
  <action>
Add the following to `src/lib/db/schema.ts`:

1. **Add `integer` import** to the `drizzle-orm/pg-core` import (already has pgTable, text, timestamp, boolean, index). Also add `unique` import for composite unique constraints.

2. **Add `isJamHost` column to user table:**
   ```
   isJamHost: boolean("is_jam_host").default(false).notNull(),
   ```

3. **Add `jamSessions` table:**
   - id: text('id').primaryKey()
   - hostId: text('host_id').notNull().references(() => user.id, { onDelete: 'cascade' })
   - scheduledAt: timestamp('scheduled_at', { withTimezone: true }).notNull()
   - location: text('location').notNull()
   - capacity: integer('capacity').notNull()
   - level: text('level').notNull() — values: 'beginner' | 'intermediate' | 'advanced' | 'all'
   - notes: text('notes') — nullable
   - createdAt: timestamp('created_at').defaultNow().notNull()
   - Indexes: on scheduledAt, on hostId

4. **Add `jamAttendees` table:**
   - id: text('id').primaryKey()
   - jamId: text('jam_id').notNull().references(() => jamSessions.id, { onDelete: 'cascade' })
   - userId: text('user_id').notNull().references(() => user.id, { onDelete: 'cascade' })
   - status: text('status').notNull() — values: 'confirmed' | 'waitlist' | 'cancelled'
   - joinedAt: timestamp('joined_at').defaultNow().notNull() — FIFO ordering key
   - Indexes: on jamId, on userId
   - Unique constraint: unique('jam_attendees_unique').on(table.jamId, table.userId) — prevents double RSVP

5. **Add `conversations` table:**
   - id: text('id').primaryKey()
   - participantA: text('participant_a').notNull().references(() => user.id, { onDelete: 'cascade' })
   - participantB: text('participant_b').notNull().references(() => user.id, { onDelete: 'cascade' })
   - createdAt: timestamp('created_at').defaultNow().notNull()
   - lastMessageAt: timestamp('last_message_at').defaultNow().notNull()
   - Indexes: composite on (participantA, participantB)
   - Unique constraint: unique('conversations_pair_unique').on(table.participantA, table.participantB) — prevents duplicate conversations
   - NOTE: Application layer enforces participantA < participantB (sorted alphabetically)

6. **Add `directMessages` table:**
   - id: text('id').primaryKey()
   - conversationId: text('conversation_id').notNull().references(() => conversations.id, { onDelete: 'cascade' })
   - senderId: text('sender_id').notNull().references(() => user.id, { onDelete: 'cascade' })
   - text: text('text').notNull()
   - sentAt: timestamp('sent_at').defaultNow().notNull()
   - Indexes: on conversationId, on sentAt

7. **Add `conversationReads` table:**
   - conversationId: text('conversation_id').notNull().references(() => conversations.id, { onDelete: 'cascade' })
   - userId: text('user_id').notNull().references(() => user.id, { onDelete: 'cascade' })
   - lastReadAt: timestamp('last_read_at').defaultNow().notNull()
   - Primary key: Use composite primary key via `.primaryKey()` on both fields, OR use a unique constraint + separate id. Simplest: add a `unique('conversation_reads_unique').on(table.conversationId, table.userId)` and a text id primary key.

8. **Add relations** (AFTER all table declarations to avoid forward-reference errors per Phase 3 decision):
   - `jamSessionsRelations`: host -> user (one), attendees -> jamAttendees (many)
   - `jamAttendeesRelations`: jam -> jamSessions (one), user -> user (one)
   - `conversationsRelations`: participantAUser -> user (one), participantBUser -> user (one), messages -> directMessages (many)
   - `directMessagesRelations`: conversation -> conversations (one), sender -> user (one)
   - Update existing `userRelations` to add: jamsHosted -> jamSessions (many), jamAttendances -> jamAttendees (many), conversationsAsA -> conversations (many), conversationsAsB -> conversations (many)

Important: Use the same patterns as existing schema — text ids, .defaultNow() timestamps, index() for foreign keys.
  </action>

  <done>schema.ts contains all 5 new tables (jamSessions, jamAttendees, conversations, directMessages, conversationReads) plus isJamHost on user, with correct types, indexes, unique constraints, and relations. TypeScript compiles cleanly.</done>
