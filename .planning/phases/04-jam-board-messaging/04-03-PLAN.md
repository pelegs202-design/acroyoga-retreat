   ```typescript
     .select({

       lastMessageAt: conversations.lastMessageAt,

       or(
         eq(conversations.participantA, session.user.id),
   ```

   - `limit` (number, default 50, max 100)
5. Query messages:
   ```typescript
   const conditions = [eq(directMessages.conversationId, conversationId)];
   if (before) {
     conditions.push(lt(directMessages.sentAt, new Date(before)));
   }
   
   const messages = await db
     .select({
       id: directMessages.id,
       senderId: directMessages.senderId,
       text: directMessages.text,
       sentAt: directMessages.sentAt,
     })
     .from(directMessages)
     .where(and(...conditions))
     .orderBy(desc(directMessages.sentAt))
     .limit(parsedLimit);
   ```
6. Reverse the messages array for display (oldest first, newest at bottom).
7. **Mark as read:** Upsert the conversationReads entry for this user + conversation:
   ```typescript
   // Check if entry exists
   const [existingRead] = await db
     .select()
     .from(conversationReads)
     .where(and(
       eq(conversationReads.conversationId, conversationId),

     .limit(1);

     await db.insert(conversationReads).values({

       userId: session.user.id,
       lastReadAt: new Date(),
   ```

Create `src/app/api/messages/unread/route.ts` with a GET handler.

**GET /api/messages/unread** — Total unread conversation count:
1. Authenticate. Return 401 if no session.
2. Count conversations where the user has unread messages. A conversation has unreads if:
   - User is a participant (participantA or participantB)
   - There exists a directMessage in that conversation where:
     - senderId !== current user (don't count own messages)
     - sentAt > user's lastReadAt for that conversation (or lastReadAt is null — never opened)
3. Implementation approach — count distinct conversations with unread messages:
   ```typescript
   import { sql } from 'drizzle-orm';
   
   // Use a raw SQL count for efficiency — this runs on every poll cycle
   const result = await db.execute(sql`
     SELECT COUNT(DISTINCT c.id) as count
     FROM conversations c
     INNER JOIN direct_messages dm ON dm.conversation_id = c.id
     LEFT JOIN conversation_reads cr
       ON cr.conversation_id = c.id AND cr.user_id = ${session.user.id}
     WHERE (c.participant_a = ${session.user.id} OR c.participant_b = ${session.user.id})
       AND dm.sender_id != ${session.user.id}
       AND (cr.last_read_at IS NULL OR dm.sent_at > cr.last_read_at)
   `);
   ```
   
   Note: Use raw SQL via `db.execute(sql`...`)` for this complex join query. The Drizzle ORM builder syntax for this multi-table join with OR + IS NULL conditions would be verbose and harder to read. Raw SQL is appropriate for aggregation queries.

4. Parse the count from the result. Neon HTTP driver returns `{ rows: [{ count: "N" }] }` or similar — check the actual shape and parseInt.
5. Return `{ unreadCount: number }`.

This endpoint will be polled every 5-10 seconds by the header component for the unread badge. Keep it efficient — single query, no joins beyond what's needed.
