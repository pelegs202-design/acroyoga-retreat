**A) Create `src/components/effects/MagneticWrapper.tsx`:**
"use client" component wrapping children with magnetic cursor pull effect. Desktop-only (uses onMouseMove/onMouseLeave). Props: `children`, `strength` (default 0.3). Uses Framer Motion `motion.div` with spring transition (stiffness: 150, damping: 15, mass: 0.1). Follow the pattern from 10-RESEARCH.md Pattern 4.

**B) Create `src/components/effects/ScrollReveal.tsx`:**
"use client" component using Framer Motion `whileInView`. Props: `children`, `className?`, `delay?` (default 0). Initial: opacity 0, y 40. Animate: opacity 1, y 0. Viewport: `{ once: true, margin: "-80px" }`. Transition: duration 0.6, ease "easeOut", delay from prop.

**C) Create `src/components/effects/ParallaxLayer.tsx`:**
"use client" component using `useScroll` + `useTransform`. Props: `children`, `speed` (default 0.4), `className?`. Wraps a container with a `ref`, `scrollYProgress` targets the ref. Background translates at `speed` ratio. Desktop only — render children directly without parallax on mobile (use a `useMediaQuery` or `hidden md:block` approach). Follow 10-RESEARCH.md Pattern 3.

**D) Create `src/components/effects/HorizontalScrollSection.tsx`:**
"use client" component for sticky horizontal scroll. Props: `children` (the items to scroll), `className?`. Uses tall outer container (h-[400vh]), sticky inner (h-screen), `useScroll` + `useTransform` to map scrollYProgress to x translation. Hidden on mobile (`hidden md:block`). On mobile, render children as a vertical stack instead. Follow 10-RESEARCH.md horizontal scroll pattern.

**E) Create `src/components/effects/DraggableCard.tsx`:**
"use client" component providing a draggable card element per DSGN-01 requirement ("at least one draggable UI element on key pages"). Uses Framer Motion `motion.div` with `drag` prop enabled. Props: `children`, `className?`, `dragConstraints?` (ref or pixel bounds — default to parent element via `dragConstraints={{ top: 0, left: 0, right: 0, bottom: 0 }}` which snaps back to origin), `dragElastic?` (default 0.2 for rubbery feel).

Implementation details:
- Wrap children in `motion.div` with `drag={true}`, `dragConstraints`, `dragElastic`, `dragTransition={{ bounceStiffness: 300, bounceDamping: 20 }}` for snappy rubber-band return
- Add `whileDrag={{ scale: 1.05, cursor: "grabbing" }}` for tactile feedback
- Default cursor: `cursor-grab` when idle, `cursor-grabbing` while dragging
- Apply `card-hover` class from globals.css for consistent glow on hover
- Desktop only: on mobile (below md), render children without drag (static card) to avoid interfering with scroll. Use a `useMediaQuery` hook or render two versions with `hidden md:block` / `md:hidden`.
- Respect reduced motion: when `prefers-reduced-motion` is active, disable the scale animation on drag but keep drag functional (drag is an interaction, not an animation)

This satisfies the ROADMAP success criterion: "at least one draggable UI element on key pages." Plan 10-02 will wire it into the homepage.
