**A) Install Radix Dialog dependency (DSGN-04 compliance):**
Run `npm install @radix-ui/react-dialog` — this provides the accessible Dialog primitive needed for the mobile menu Sheet pattern. DSGN-04 requires "shadcn/ui + Radix UI (accessible primitives)." Since the project does not use shadcn's CLI, install the Radix primitive directly and build the Sheet pattern manually. This gives us focus trap, escape-to-close, scroll lock, and proper ARIA attributes out of the box.

**B) Redesign `src/components/layout/Header.tsx`:**
Keep ALL existing functionality (session check, unread polling, sign-out, language toggle, nav links). Redesign the visual layer:

- Sticky top-0 z-50, but with BOLDER brutalist styling: thicker bottom border (border-b-2 border-brand or border-neutral-700), slightly taller (h-18 or h-20)
- Brand wordmark: larger (text-2xl), Heebo 900, tracking-tighter, text-brand — make it the dominant visual anchor
- Desktop nav links: wrap each in MagneticWrapper for magnetic cursor pull. Style them bolder: uppercase, tracking-wide, text-sm font-bold, with a subtle hover underline effect (using `after:` pseudo-element that scales from 0 to full width on hover, bg-brand)
- Unread badge: keep pink pill style
- Sign-out/Sign-in button: add `btn-press` class for press feedback
- Mobile: hide desktop nav links. Add a hamburger button (3 horizontal lines icon) that triggers MobileMenu open state. Pass `isOpen` and `onOpenChange` to MobileMenu.
- All existing i18n keys preserved. Add `common.menu` key for "Menu" if needed.

**C) Create `src/components/layout/MobileMenu.tsx`:**
"use client" component built on Radix Dialog for WCAG AA accessibility (DSGN-04). Props: `isOpen: boolean`, `onOpenChange: (open: boolean) => void`.

Implementation using Radix Dialog primitives:
```
import * as Dialog from "@radix-ui/react-dialog";
```
- Use `Dialog.Root` with `open={isOpen}` and `onOpenChange={onOpenChange}` for controlled state
- Use `Dialog.Portal` to render outside the DOM hierarchy
- Use `Dialog.Overlay` with a semi-transparent dark backdrop (bg-black/60, backdrop-blur-sm)
- Use `Dialog.Content` as the slide-in panel: fixed position, slides in from the end (right in LTR, left in RTL — use `inset-inline-end: 0`), width 80vw max-w-sm, bg-neutral-900, border-s-2 border-neutral-800, full height
- Use `Dialog.Title` (visually hidden via sr-only) with "Navigation menu" for screen readers
- Use `Dialog.Close` for the X close button at the top of the panel
- Navigation links inside: Members, Jams, Messages (with unread badge), Profile Edit, Settings, Sign Out — all using Link from @/i18n/navigation
- Close on link click by calling `onOpenChange(false)` in the link onClick handler
- RTL-safe using CSS logical properties (inset-inline-end, border-inline-start, etc.)

This gives us FOR FREE from Radix:
- Focus trap (Tab cycles within menu)
- Escape key closes menu
- aria-modal="true" on content
- Scroll lock on body when open
- Proper focus restoration to trigger button on close

Add a CSS transition or Framer Motion `AnimatePresence` for smooth slide-in/out. Keep it simple — the locked decision says "standard dropdown, no special animation" but a basic slide transition is standard UX, not special animation. Use a 200ms ease-out slide.

- Copyright: "2026 AcroHavura" from i18n
- Layout: use CSS grid or flex. On mobile: stacked. On desktop: 3-column (brand, nav, social)
- RTL-safe with logical properties
