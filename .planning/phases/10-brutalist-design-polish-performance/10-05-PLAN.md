<task type="auto">
  <name>Task 2: Lighthouse performance audit and optimization</name>
  <files>

  <action>
**A) Image optimization audit:**
Search the entire `src/` directory for `<img` tags not using next/image. Replace any found with `<Image>` from "next/image" with proper `width`/`height` or `fill` + sized container. For hero/LCP images, use `preload={true}` (NOT priority, which is deprecated in Next.js 16). For all images, set `sizes` prop for responsive loading. Quality: hero=90, general=75.

**B) Animation performance audit:**
Grep all Framer Motion usage across `src/`. Verify NO animation targets layout-triggering properties (width, height, top, left, padding, margin). Only `transform` (x, y, scale, rotate) and `opacity` are allowed. Fix any violations found.

Check that `will-change: transform` is NOT applied globally — only on actively animating elements.

**C) Font loading optimization:**
Check `src/app/[locale]/layout.tsx` — Heebo is already loaded via `next/font/google` with `display: "swap"` and `preload: true`. Verify the `subsets` include both "latin" and "hebrew". If weights include unused ones, consider trimming (but Heebo 400, 500, 700, 900 are all used — keep them).

**D) Mobile animation reduction:**
Review HorizontalScrollSection and ParallaxLayer. Verify:
- HorizontalScrollSection is hidden on mobile (`hidden md:block` or similar) with vertical fallback
- ParallaxLayer either disables parallax on mobile or renders children directly
- No `useScroll` hooks fire on mobile (they should only exist in desktop-only components)

**E) Build and measure:**
Run `npm run build` — check output for:
- Bundle size warnings (any page JS > 200KB is a concern)
- Static/dynamic page status
- Any build errors

If possible, run `npx next build --experimental-build-mode generate` or similar to check page sizes.

**F) Additional optimizations if needed:**
- Ensure 3rd-party scripts (GA4, Meta Pixel) remain `strategy="afterInteractive"` (already confirmed in layout.tsx)
- Check for any large inline styles or data URIs that inflate HTML payload
- Verify `blurDataURL` values (if any) are small (< 500 bytes)
- Add dynamic imports for heavy client components if bundle is too large (e.g., `dynamic(() => import('./HorizontalScrollSection'), { ssr: false })` for the horizontal scroll section which is desktop-only anyway)

**G) Performance budget documentation:**
Create a brief comment block at the top of `next.config.ts` documenting the performance targets:
```
// Performance budget (Phase 10 — Lighthouse 90+ mobile):
// LCP < 2.5s, CLS < 0.1, FCP < 1.8s, TBT < 200ms
// Animate ONLY: transform, opacity (compositor-safe)
// Hero image: preload={true}, quality=90
// Mobile: no parallax, no horizontal scroll, no useScroll
```
