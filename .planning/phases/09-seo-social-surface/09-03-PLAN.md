1. Create `src/lib/instagram.ts`:
   - Export `IgPost` type: `{ id, caption?, media_url, permalink, thumbnail_url?, timestamp, media_type: 'IMAGE' | 'VIDEO' | 'CAROUSEL_ALBUM' }`
   - Export `fetchInstagramFeed(limit = 9): Promise<IgPost[]>`:
     - Reads `INSTAGRAM_ACCESS_TOKEN` from `process.env` (NOT `NEXT_PUBLIC_` -- server only)
     - If token is missing, return empty array (graceful degradation -- no crash)
     - Fetches `https://graph.instagram.com/me/media?fields=id,caption,media_url,permalink,thumbnail_url,timestamp,media_type&limit=${limit}&access_token=${token}`
     - Uses `{ next: { revalidate: 21600 } }` for 6-hour ISR cache
     - On error, log `[Instagram] Feed fetch failed: {status}` and return empty array
     - Return `data.data ?? []`
   - Export `refreshInstagramToken(): Promise<string | null>`:
     - Calls `https://graph.instagram.com/refresh_access_token?grant_type=ig_refresh_token&access_token=${token}`
     - Returns new access_token string or null on failure
     - This will be called by an admin route or cron (not in this plan)
   - Follow Pattern 4 from 09-RESEARCH.md exactly

2. Create `src/components/social/InstagramGrid.tsx` (Server Component):
   - Imports and calls `fetchInstagramFeed(9)` directly (RSC -- no useEffect)
   - If posts array is empty, render nothing (hidden section -- no broken UI)
   - Renders a grid: 3 columns on desktop (lg:grid-cols-3), 3 columns on mobile too (grid-cols-3) for a 3x3 or 3x2 layout
   - Each post is an anchor tag (`<a href={post.permalink} target="_blank" rel="noopener">`) wrapping:
     - For IMAGE/CAROUSEL_ALBUM: `<img>` with `media_url`, lazy loading, aspect-square crop
     - For VIDEO: `<img>` with `thumbnail_url` fallback, play icon overlay
   - Brutalist style: dark border-2, slight hover effect (scale or brightness), no rounded corners
   - Section heading: "Follow us on Instagram" / "עקבו אחרינו באינסטגרם" + link to @acroshay profile

   - Add a section wrapper with appropriate spacing and heading
