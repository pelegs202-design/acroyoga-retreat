1. Install new packages: `npm install schema-dts react-share`

2. Create `src/lib/seo/metadata.ts` -- a `buildPageMetadata` async function that:
   - Accepts `{ locale, namespace, path }` params
   - Uses `getTranslations({ locale, namespace })` from `next-intl/server` to read title + description
   - Sets BASE_URL to `https://acroretreat.co.il`
   - Returns Next.js `Metadata` object with:
     - title, description from translations
     - `alternates.canonical` (locale-aware)
     - `alternates.languages` with `he`, `en`, and `x-default` (Hebrew as x-default per locked decision)
     - `openGraph` with title, description, siteName 'AcroHavura', locale (he_IL / en_US), alternateLocale, images array pointing to `/og-image.jpg` (1200x630), type 'website'
     - `twitter.card` = 'summary_large_image' with image
   - Follow the exact pattern from 09-RESEARCH.md Pattern 1

3. Create `src/lib/seo/schemas.ts` -- typed JSON-LD schema builders using `schema-dts`:
   - `buildOrganizationSchema()` -- returns `WithContext<Organization>` with name 'AcroHavura', alternateName Hebrew, sameAs Instagram @acroshay, bilingual contact point
   - `buildLocalBusinessSchema(city: 'tel-aviv' | 'kfar-saba')` -- returns `WithContext<LocalBusiness>` with city-specific Hebrew names and addresses (Tel Aviv / Kfar Saba)
   - `buildEventSchema(jam)` -- returns `WithContext<Event>` with Hebrew jam name, start date, location, organizer
   - `buildFAQSchema(faqs: Array<{ q: string; a: string }>)` -- returns `WithContext<FAQPage>` with Question/Answer entities
   - Follow the exact patterns from 09-RESEARCH.md

4. Create `src/components/seo/JsonLd.tsx` -- a generic RSC component:
   - Accepts `{ data: WithContext<T> }` where T extends `Thing` from schema-dts
   - Serializes to JSON with standard JSON-in-script-tag sanitization (replace angle brackets with unicode escapes)
   - Renders a script tag with type application/ld+json (this is the official Next.js JSON-LD pattern from nextjs.org/docs/app/guides/json-ld -- the data is controlled server-side schema output, not user input)

5. Add i18n keys to `messages/he.json` and `messages/en.json`:
   - `seo.home.title` / `seo.home.description` -- homepage meta using Hebrew acroyoga keywords from research (use the recommended meta descriptions from 09-RESEARCH.md)
   - `seo.cityTelAviv.title` / `seo.cityTelAviv.description` -- Tel Aviv city page meta
   - `seo.cityKfarSaba.title` / `seo.cityKfarSaba.description` -- Kfar Saba city page meta
   - Use the recommended Hebrew meta descriptions from 09-RESEARCH.md verbatim
